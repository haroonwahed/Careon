# Frontend UX Skill

Use this skill for UI, UX, navigation, copy, layout, and page/component work.

## Rules

- Dutch-first user-facing copy.
- Every page must answer: waar staat de casus, waarom, wat blokkeert, wie is eigenaar, wat is de volgende actie.
- Prefer action banners, clear CTAs, progress states, evidence panels, and blocker messages.
- Avoid generic dashboard widgets unless they drive a concrete operational action.
- Keep UI calm, professional, modern, and high-signal.
- Do not replace domain terms with generic English.
- Do not create dead-end pages.

## Before editing

- identify the active route/page/component
- check if the component is actually rendered
- avoid polishing orphaned/legacy surfaces unless explicitly requested
