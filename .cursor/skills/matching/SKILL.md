# Matching Skill

Use this skill for provider recommendations, confidence scores, factor breakdowns, and rejection learning.

## Matching must:

- explain why providers are suggested
- show trade-offs
- preserve confidence calibration
- use care type, urgency, region, capacity, specialization, complexity, and special needs
- capture rejection reasons structurally
- avoid opaque ranking labels

## Never:

- present matching as final placement
- claim certainty when data is weak
- hide weak-match warnings
- drop auditability for smoother UX
