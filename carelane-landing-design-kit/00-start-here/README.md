# Carelane Landing Page Design Kit v2.0

This package is the implementation handoff for the Carelane public landing page.

## Purpose

Build a premium, calm and operational landing page for Carelane. The page must communicate that Carelane is the control layer for care allocation under scarcity. It is not a generic dashboard, feature catalogue or AI marketing page.

## Start here

1. Read `01-agent-handoff/MASTER_AGENT_PROMPT.md`.
2. Treat the two documents in `08-source-of-truth/` as binding.
3. Inspect the reference images in `07-reference-images/`.
4. Implement the page using the tokens in `02-design-system/`.
5. Follow the section contract in `03-page-specification/`.
6. Run the acceptance checklist in `06-quality-assurance/`.

## Required section order

1. Navigation
2. Hero
3. Care journey route
4. Regiekamer intelligence
5. One case, multiple organisations
6. Explainable matching
7. Trust by design
8. Results in practice
9. Built for each role
10. Final CTA
11. Footer

## Non-negotiable workflow

Aanmelding → Matching → Aanbiederreactie → Plaatsing → Intake

Do not add Samenvatting, Beoordeling or Gemeente Validatie as top-level phases.

## Brand

Use **Carelane** everywhere. Remove remaining references to CareOn, Zorg OS or other legacy names from the landing page.
