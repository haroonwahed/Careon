# Responsive behaviour

## Desktop: 1280 px and above

- 12-column grid
- Split hero, 5 columns copy and 7 columns product preview
- Workflow phases in one connected horizontal route
- Network visual centred with surrounding nodes
- Matching comparison uses three cards plus explanation area

## Tablet: 768 to 1279 px

- 8-column grid
- Hero stacks copy above product preview around 900 to 1024 px when needed
- Workflow route may wrap into a controlled 3 + 2 layout while preserving order and connection
- Network nodes use a two-column orbit or structured list
- Matching cards scroll horizontally only when accessibility and discoverability remain clear; a stacked layout is preferred

## Mobile: below 768 px

- Single-column flow
- No horizontal page overflow
- Navigation becomes an accessible drawer
- Hero product preview remains readable, not a tiny desktop screenshot
- Workflow becomes a vertical timeline
- Network becomes a central case followed by role cards and directional connectors
- Matching cards stack vertically
- Trust pillars become compact rows
- Footer columns collapse into accordions or clear link groups

## Required widths to verify

- 390 px
- 768 px
- 1024 px
- 1440 px

## Touch and readability

- Minimum target size: 44 by 44 px
- Body text minimum: 16 px
- Never rely on hover for essential information
- Avoid text smaller than 12 px in product previews
