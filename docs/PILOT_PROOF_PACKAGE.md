# Pilot proof package (rehearsal artifacts)

**Audience:** operators, release reviewers, and agents working on infrastructure maturity—not end-user documentation.

**Scope:** Evidence produced by **`./scripts/run_full_pilot_rehearsal.sh`** under `reports/` after a deterministic rehearsal database reset. This package answers whether the **rehearsal stack** reproduces expected tenancy, workflow-adjacent APIs, and the **Case Timeline v1 boundary** from gemeente validatie through aanbieder beoordeling.

---

## 0. Release ritual (default cadence)

Treat the pilot rehearsal as **part of shipping**, not an optional extra:

| When | What |
|------|------|
| **Every merge to `main`** | Full Django pytest + guardrails already run in GitHub Actions (`Platform Guardrails`). |
| **Weekly + on `v*` tags + manual** | Workflow **`Pilot rehearsal (release evidence)`** (`.github/workflows/pilot-rehearsal.yml`) runs `./scripts/run_full_pilot_rehearsal.sh` and **uploads** the `reports/pilot-<run_id>/` bundle as a workflow artifact (retention 180 days). Download the artifact for GO/NO-GO review. |
| **Manual + Playwright (optional)** | In Actions → **Run workflow**, enable input **`with_playwright`** to append `--with-playwright` (SPA build + browser golden path; slower). Scheduled/tag runs keep the default headless/TestClient path. |
| **Before pilot / prod promotion** | Run the script locally or via workflow_dispatch; inspect `release_evidence_bundle.json` and merged `rehearsal_report.json` per sections below. |

Local output lives under `reports/` (gitignored). CI artifacts are the **archived** chain of evidence for auditors and release managers.

## 1. Purpose

| Goal | What the package supports |
|------|---------------------------|
| **Deterministic pilot verification** | Same seed path (`reset_pilot_environment`) + same commands → comparable JSON for regression review. |
| **Operational traceability** | Machine-readable artifacts record check names, HTTP outcomes, and timeline facts without relying on screenshots. |
| **Release readiness evidence** | `release_evidence_bundle.json` records whether the timeline gate is **GO** or **NO-GO** for pilot sign-off during infrastructure maturity. |
| **Workflow integrity verification** | `rehearsal_verify` asserts seeded tenant shape, routes, API smoke, placements, escalation lane—not a substitute for full workflow tests, but a tight rehearsal contract. |
| **Provider visibility enforcement** | Timeline evidence records gemeente vs linked provider vs unrelated provider HTTP outcomes on the timeline endpoint. |
| **Timeline causality proof** | Ordered timeline event types show the intended sequence from validation approval through placement request to provider review opened—not assignment semantics; matching remains advisory per product rules. |

The package is **not** a product feature list. It is **operational proof** that the rehearsal pipeline and timeline boundary checks succeeded on the run that produced these files.

## 2. How to generate

From repository root (rehearsal settings and DB are script defaults unless you override env):

```bash
./scripts/run_full_pilot_rehearsal.sh
```

Optional flags (see script header): `--with-playwright` (auto-starts `runserver` on `E2E_BASE_URL` when omitted), `--start-server`, `--skip-spa-build`, `--http-preflight`.

With Playwright, `run_golden_path_e2e.sh` runs **`zorg-os-golden-path`**, **`staging-shell-smoke`**, and **`provider-review-smoke`** (reject submit last; rehearsal DB only).

**Environment (common):**

- `DJANGO_SETTINGS_MODULE` — defaults to `config.settings_rehearsal`.
- `REPORT_DIR` — override output directory; default `reports/` under repo root.
- `VERIFY_LOG` — override path for `rehearsal_verify` human-oriented log; default `reports/rehearsal_verify.log`.

Use **`config.settings_rehearsal`** and the rehearsal SQLite file expected by your team—do not mix production DB URLs into this path unless that is an explicit operational decision.

## 5. JSON integrity

Quick check that `rehearsal_report.json` is a **single valid JSON document** (no trailing prose):

```bash
python -m json.tool reports/rehearsal_report.json >/dev/null
```

Repeat for other JSON artifacts as needed:

```bash
python -m json.tool reports/rehearsal_timeline_evidence.json >/dev/null
python -m json.tool reports/release_evidence_bundle.json >/dev/null
```

Exit code **0** means parse succeeded.

## 6. Timeline boundary evidence

### Expected event order

The release gate expects **`event_types_ordered`** exactly:

1. `GEMEENTE_VALIDATION_APPROVED`
2. `PLACEMENT_REQUEST_CREATED`
3. `PROVIDER_REVIEW_OPENED`

That order reflects **operational causality** along the canonical path from gemeente validatie toward aanbieder beoordeling—not an exhaustive timeline of all future event types.

### Fields (what rehearsal checks)

| Concern | Evidence fields (typical) | Meaning |
|---------|---------------------------|---------|
| **Request correlation** | `request_ids_present` | Request identifiers appear where required for traceability checks. |
| **Metadata safety** | `metadata_keys_ok` | Timeline row metadata stays within allowed operational keys (no PHI drift in rehearsal assertion). |
| **Gemeente access** | `gemeente_timeline_status` | Expected **200** for gemeente caller on case timeline API. |
| **Linked provider access** | `linked_provider_timeline_status` | Expected **200** for the assigned provider on the same case. |
| **Unrelated provider denial** | `unrelated_provider_timeline_status` | Expected **404** for a provider not entitled to that case’s timeline. |

Detailed semantics of timeline rows are documented in **[CASE_TIMELINE_V1.md](CASE_TIMELINE_V1.md)**.

## 7. GO / NO-GO semantics

Interpret **`reports/release_evidence_bundle.json`** → **`timeline_gate`**:

### GO (pilot timeline readiness)

Requires, together:

- Loadable timeline evidence (`ok: true` from merged sources).
- **`timeline_gate.go`** is **true** (and usually **`no_go_reasons`** empty).

Concretely, the gate checks evidence for: correct **event order**, **`request_ids_present`**, HTTP outcomes above, **`metadata_keys_ok`**, and **`authorization_checks_passed`**. The rehearsal script fails if **`manage.py release_evidence_bundle`** exits non-zero after a successful merge.

### NO-GO (examples)

| Situation | Typical signal |
|-----------|----------------|
| Corrupted or non-parseable JSON | `json.tool` fails; merge or bundle commands fail. |
| Missing evidence files | `timeline_evidence_missing` or missing files on disk. |
| Timeline run failed | `ok: false` or evidence absent. |
| Wrong order | `event_order_invalid` in `no_go_reasons`. |
| Provider visibility wrong | `gemeente_timeline_access_failed`, `linked_provider_timeline_access_failed`, `unrelated_provider_denial_failed`. |
| Request IDs | `request_ids_present_false`. |
| Metadata | `metadata_safety_failed`. |
| Authorization bundle | `authorization_checks_failed`. |

During **infrastructure maturity**, treat **NO-GO** as **stop and investigate** before declaring pilot-ready—unless an explicit, documented exception applies (see below).

## 8. `SKIP_TIMELINE_RELEASE_GATE=1`

Used by **`scripts/production_readiness_gates.sh`** Gate 8 only.

| Topic | Guidance |
|-------|----------|
| **What it does** | Skips the **`manage.py release_evidence_bundle`** gate when set to **`1`**. |
| **Why it exists** | Allows environments (e.g. fresh CI without rehearsal artifacts) to run other gates without blocking on missing `reports/` data. |
| **When it is acceptable** | Narrowly: CI jobs that only validate migrations/tests/seeds; **not** for declaring pilot GO or production readiness with timeline proof. |
| **Why it must stay exceptional** | Pilot **GO** for the timeline boundary is grounded in **actual rehearsal outputs**. Skipping the gate removes that check from the pass/fail surface—only use with team visibility. |

Default for pilot sign-off: **do not set** this variable.

## 9. Troubleshooting

| Symptom | Likely cause | Direction |
|---------|----------------|-----------|
| `JSONDecodeError: Extra data` on `rehearsal_report.json` | File contained non-JSON after the first object (legacy `tee` / extra stdout). | Regenerate with current **`run_full_pilot_rehearsal.sh`**; validate with **`json.tool`**. |
| Missing `release_evidence_bundle.json` | Script failed before Step 5, or `release_evidence_bundle` not run. | Re-run full rehearsal; inspect stderr and **`rehearsal_verify.log`** / **`rehearsal_timeline_step.log`**. |
| Unrelated provider returned **200** instead of **404** | Visibility/regression on timeline route for wrong tenant/provider. | Treat as **NO-GO**; investigate authorization and case–provider linkage—see timeline tests and API guards. |
| Missing timeline evidence | `rehearsal_timeline_evidence` did not run or wrote elsewhere (`REPORT_DIR`). | Confirm paths; run `manage.py rehearsal_timeline_evidence --json-out reports/rehearsal_timeline_evidence.json` after seed. |
| Release gate failure with reasons in bundle | Gate lists concrete `no_go_reasons`. | Fix underlying evidence fields; re-run rehearsal; confirm **`timeline_gate.go`** true. |
