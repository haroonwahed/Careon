# Friday Facilitator Script - 2026 W16

Session length: 90 minutes  
Lead: Team Lead / Reviewer  
Participants: Regiekamer Lead, Matching Operator, Care Coordinator, Product/Ops Support

## Opening Script (read out loud, 2 minutes)
Today is a structured learning session.
We are not proving the system is perfect.
We are proving decisions can be reviewed, explained, and improved.
No blame, no side debates, no skipping classification.

## Timebox Plan
- 0 to 5 min: setup and rules
- 5 to 65 min: case review loop (5 to 6 cases)
- 65 to 80 min: pattern check
- 80 to 90 min: close questions and action ownership

## Rule Set (read out loud)
- Facts first, opinions second.
- Classification before discussion.
- One primary reason per case.
- Move on when timebox ends.

## Case Review Loop Script (repeat per case)
Step 1: Show facts only
- What did the system recommend?
- What action did we take?
- What was the outcome?

Step 2: Ask one question
- Given what we knew at the time, was this the right decision?

Step 3: Force one classification
- SYSTEM_CORRECT
- USER_CORRECT
- BOTH_ACCEPTABLE
- BOTH_SUBOPTIMAL

Step 4: Force one reason
- missing data
- provider mismatch
- capacity issue
- SLA timing
- explanation unclear
- external constraint

Step 5: Move on
- Capture evidence sentence and continue.

## Pattern Check Questions (10 to 15 minutes)
Ask exactly:
- What did we see multiple times?
- Where did the system struggle?
- Where did we struggle?

Capture only recurring patterns.

## Close Questions (5 minutes)
Ask exactly:
- Would you actually use this weekly?
- What felt unclear or annoying?
- What would make this 10x more useful?

## Facilitator Observation Checklist
Watch for:
- hesitation to classify
- confusion between SYSTEM_CORRECT and USER_CORRECT
- vague reasons
- skipped replay evidence
- emotional argument replacing structured reasoning

## Immediate Fix Threshold
Only escalate immediate fixes if:
- categories are misunderstood
- output is confusing in-session
- replay cannot explain decisions
- command output is unusable for discussion

## Noise Filter
Ignore for now:
- UI polish requests
- new AI ideas
- automation wishes
- extra filter requests

## Session Success Gate
Session is successful if:
- classifications are completed for all reviewed cases
- each case has one reason and one evidence sentence
- recurring patterns are captured
- follow-up actions have owner and due date
