# CareOn Page Patterns

This document describes the approved page patterns for CareOn UI surfaces.
It is subordinate to `CAREON_UI_CONTRACT.md`.

## Approved Patterns

- Operational page
  - page title
  - optional subtitle
  - one dominant CTA
  - one attention surface maximum
  - one workvoorraad section maximum

- Detail page
  - page title
  - context summary
  - one primary action
  - one supporting actions area

- Shell page
  - sidebar
  - topbar
  - content area with contract padding and radius

## Reuse Rules

- Reuse existing primitives before creating a new visual pattern.
- Do not introduce a new card, badge, table, or CTA style if an existing component already covers the need.
- If a page needs a new pattern, update the contract and shared primitives first.
